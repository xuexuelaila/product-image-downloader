# 商品标题抓取兜底策略

## 📋 问题背景

电商网站的页面结构可能随时变化，导致商品标题抓取失败。为了确保程序的稳定性，我们实现了**多层兜底策略**，确保在任何情况下都能获取到一个可用的标题。

## 🎯 抓取策略（优先级从高到低）

### 方法1: JavaScript 变量提取（优先级最高）✨

**原理**: 从页面 JavaScript 中提取 `name: '商品名称'` 字段

**优点**:
- 最准确，直接来自商品数据
- 不受页面 DOM 结构变化影响
- 包含完整的商品标题

**代码**:
```javascript
const scripts = document.querySelectorAll('script');
for (let script of scripts) {
    const match = script.textContent.match(/name:\s*'([^']+)'/);
    if (match && match[1] && match[1].length > 10) {
        return match[1];
    }
}
```

**示例输出**:
```
追觅X60Pro【春晚合作扫地机】扫地机器人扫拖洗烘一体自清洁滚筒洗地机器人热水洗拖布水箱版RLX72DE
```

**失败场景**:
- 页面 JavaScript 结构变化
- 商品数据格式改变
- 页面加载不完整

---

### 方法2: DOM 选择器提取（备用方案）🔄

**原理**: 使用多个 CSS 选择器尝试匹配商品标题元素

**选择器列表**:
```python
[
    ".sku-name",
    ".itemInfo-wrap .sku-name",
    "div.sku-name",
    "h1",
    "[class*='sku-name']",
    "[class*='product-name']",
    "[class*='item-name']"
]
```

**验证规则**:
- 标题长度 > 10 字符
- 不包含"只换不修"、"全保换新"等服务选项文本

**示例输出**:
```
追觅X60Pro扫地机器人
```

**失败场景**:
- DOM 结构大幅改版
- 所有选择器都匹配不到
- 匹配到的文本不符合验证规则

---

### 方法3: 页面 Title 提取（兜底方案1）🛡️

**原理**: 从页面 `<title>` 标签中提取商品标题，并清理平台后缀

**清理规则**:
```python
# 移除京东特有的后缀
title = page_title.split("【图片")[0]  # 移除 "【图片 价格 品牌 报价】"
               .split("-京东")[0]      # 移除 "-京东"
               .split("_")[0]          # 移除其他分隔符
               .strip()
```

**示例**:
```
原始: 追觅X60Pro【春晚合作扫地机】扫地机器人【图片 价格 品牌 报价】-京东
提取: 追觅X60Pro【春晚合作扫地机】扫地机器人
```

**优点**:
- 几乎所有页面都有 title 标签
- 通常包含商品核心信息
- 提取成功率高

**失败场景**:
- title 标签为空或无效
- 清理后标题太短（< 10 字符）

---

### 方法4: 商品 ID 提取（兜底方案2）🆔

**原理**: 从 URL 中提取商品 SKU ID，作为标题

**提取逻辑**:
```python
# 从 URL 提取商品 ID
# 例如: https://item.jd.com/100313528034.html
match = re.search(r'/(\d+)\.html', url)
if match:
    sku_id = match.group(1)
    title = f"商品_{sku_id}"
```

**示例**:
```
URL: https://item.jd.com/100313528034.html
标题: 商品_100313528034
```

**优点**:
- 唯一标识商品
- 提取成功率极高
- 不依赖页面内容

**失败场景**:
- URL 格式不符合预期（极少见）

---

### 方法5: 时间戳（最终兜底）⏰

**原理**: 使用当前时间戳作为标题

**格式**:
```python
title = f"商品_{datetime.now().strftime('%Y%m%d_%H%M%S')}"
```

**示例**:
```
商品_20260309_193000
```

**优点**:
- 100% 成功
- 永远不会失败

**缺点**:
- 无法识别商品
- 需要手动重命名

## ⚠️ 注意事项

### 1. 标题验证
所有提取的标题都会经过验证：
- 长度 > 10 字符（除了最终兜底）
- 不包含服务选项关键词
- 去除首尾空白字符

### 2. 文件名清理
提取的标题会自动清理非法字符：
```python
safe_title = re.sub(r'[<>:"/\\|?*]', '', title)
```

### 3. 调试信息
每个方法都会输出详细的调试信息，便于排查问题：
```
  尝试从 JavaScript 中提取商品标题...
  从 JavaScript 中找到标题: ...
```

### 4. 不会抛出异常
无论什么情况，程序都会返回一个可用的标题，不会因为标题抓取失败而中断。
