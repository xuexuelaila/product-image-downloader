## 📊 测试结果分析

根据签名测试结果：
- ✅ APP_KEY 长度正确（32 位）
- ✅ APP_SECRET 长度正确（32 位）

**结论**：签名生成代码没有问题，问题很可能在于：
1. APP_KEY 或 APP_SECRET 不正确（虽然格式对，但可能是错误的密钥）
2. 应用未获得 API 调用权限
3. 应用类型不匹配（联盟应用 vs 商家应用）

## 🎯 解决步骤（按优先级）

### 步骤 1：登录京东开放平台核对密钥（最重要）

1. **访问京东开放平台**
   ```
   https://open.jd.com/
   ```

2. **登录账号**
   - 使用你的京东账号登录

3. **进入应用管理**
   - 点击右上角"控制台"
   - 选择"应用管理"
   - 找到你创建的应用

4. **核对密钥**
   - 查看应用详情
   - 找到 App Key 和 App Secret
   - **仔细对比**当前配置：
     ```
     APP_KEY: 901a3dbb017ab45074c6dfc66e269a59
     APP_SECRET: e11a771e1dfe4c80adfa9e0f71abbc7a
     ```
   - 确保**完全一致**（一个字符都不能错）

5. **如果不一致**
   - 复制正确的密钥
   - 编辑 `scripts/jd_api_config.py`
   - 粘贴正确的密钥
   - 保存文件
   - 重新测试

### 步骤 2：检查应用类型和状态

1. **应用类型**
   - 查看应用类型是否为"联盟应用"
   - 当前项目使用的是**京东联盟 API**
   - 如果不是联盟应用，需要重新创建

2. **应用状态**
   - 查看应用状态
   - 应该是"已上线"或"测试中"
   - 如果是"未上线"或"已禁用"，需要先上线

3. **加入京东联盟**
   - 京东联盟 API 需要先加入京东联盟
   - 访问：https://union.jd.com/
   - 注册成为京东联盟会员

### 步骤 3：检查 API 权限

1. **查看 API 权限**
   - 在应用详情页面
   - 找到"API 权限管理"或"接口权限"
   - 查看是否有以下权限：
     - `jd.union.open.goods.query`
     - `jd.union.open.goods.bigfield.query`

2. **申请权限（如果没有）**
   - 点击"申请权限"或"添加接口"
   - 搜索并申请上述两个 API
   - 填写申请理由（例如："用于商品信息查询"）
   - 提交申请
   - 等待审核（通常几分钟到几小时）

## 🔧 常见问题和解决方案

### 问题 1：找不到应用

**可能原因**：
- 应用被删除了
- 登录的账号不对

**解决方案**：
- 确认登录的是创建应用的账号
- 如果应用被删除，需要重新创建

### 问题 2：应用类型不对

**症状**：
- 应用类型显示为"商家应用"或其他类型
- 不是"联盟应用"

**解决方案**：
1. 重新创建应用，选择"联盟应用"类型
2. 或者使用商家 API（需要修改代码）

### 问题 3：权限申请被拒绝

**可能原因**：
- 申请理由不充分
- 账号资质不符合要求

**解决方案**：
- 完善申请理由
- 补充账号资质信息
- 联系京东客服

### 问题 4：仍然报"无效签名"错误

**如果完成所有步骤后仍然报错**：

1. **尝试重新生成密钥**
   - 在应用详情页面
   - 找到"重置 Secret"或类似选项
   - 重新生成 App Secret
   - 更新配置文件
   - 重新测试

2. **尝试不同的 API 域名**
   编辑 `scripts/jd_api_config.py`：
   ```python
   # 尝试方案 1
   API_DOMAIN = "api.jd.com"

   # 或尝试方案 2
   API_DOMAIN = "router.jd.com"
   ```

3. **联系京东技术支持**
   - 在京东开放平台提交工单
   - 描述问题：API 调用返回"无效签名"错误（错误代码 12）
   - 提供：App Key、调用的 API 名称、错误信息

## 💡 重要提示

1. **密钥安全**
   - 不要将密钥分享给他人
   - 不要将密钥提交到公开的代码仓库
   - 如果密钥泄露，立即重置

2. **API 限制**
   - 京东 API 可能有调用频率限制
   - 注意不要频繁调用
   - 如果超限，会返回其他错误

3. **爬虫模式兜底**
   - 即使 API 不工作，爬虫模式仍然可用
   - 混合模式会自动回退
   - 不影响正常使用
